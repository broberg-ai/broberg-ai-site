# F001.8 — Light mode

## Motivation
Christian wants a LIGHT theme alongside the current design (now named "Dark"). cms #238 relayed his approval + a starting palette; Christian curates colours after a first pass.

## Scope
- Theme toggle (sun/moon) in the nav, persisted to localStorage, default = `prefers-color-scheme`.
- Inline `<head>` script sets the theme before first paint (no FOUC).
- `[data-theme="light"]` on `<html>` overrides the `:root` tokens with the light palette.
- Edge-cases where a token swap is not enough: `.card` shadow, `header` bg, `.cmdk-backdrop`, and illustrations/gradients drawn for a dark ground.
- WCAG AA in both themes.

## Light palette (cms #238 starting bud — Christian curates after)
`--blue #0096db` · `--blue-light #007bb8` · `--blue-glow rgba(0,150,219,.10)` · `--dark #f5f7fa` (page bg) · `--dark2 #ffffff` (header/cmdk/footer) · `--card #ffffff` · `--card-border rgba(17,25,38,.11)` · `--light #161b22` (text) · `--muted rgba(22,29,40,.60)` · `--orange #ec4e27` · `--orange-text #cf3f1b` · `--orange-soft rgba(236,78,39,.10)`.

## Non-goals
- No per-page theme; one global toggle.
- Not re-drawing illustrations from scratch — light-variant tweaks only where they wash out.

## AC
See the card's acceptance criteria (toggle+persist+default, no-FOUC head script, `[data-theme=light]` token override, edge-cases, WCAG AA both themes, Lens-verify both themes home/post/⌘K/tags + no dark regression).
